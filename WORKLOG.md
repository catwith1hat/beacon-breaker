### 10. `process_slashings` per-epoch + `process_slashings_reset` (EIP-7251 algorithm restructure)

**Status:** no-divergence-pending-fuzzing — audited 2026-05-02. Per-epoch slashings drain; closes the `state.slashings[]` vector cycle (items #8 + #9 wrote into it; this drains and resets it).

EIP-7251 modified `process_slashings` to **restructure the per-validator penalty algorithm to reduce floor-division precision loss**. Constants are unchanged (`PROPORTIONAL_SLASHING_MULTIPLIER_BELLATRIX = 3` is still used at Electra). The change is purely algorithmic and subtle: pre-Electra computed `penalty = (effective_balance/increment * adjusted_total_slashing_balance) / total_balance * increment` per validator; Pectra computes `penalty_per_increment = adjusted_total_slashing_balance / (total_balance / increment)` once per epoch (loop-invariant), then `penalty = penalty_per_increment * effective_balance_increments` per validator. ### 16. `compute_exit_epoch_and_update_churn` + `compute_consolidation_epoch_and_update_churn` (Pectra-NEW per-block churn primitives)

**Status:** no-divergence-pending-fuzzing — audited 2026-05-02. The chokepoint Pectra primitives that ALL Track A operations (items #2/#3/#6) plus slashing-induced exits (items #8/#9) flow through. Item #6 explicitly flagged as "highest-leverage primitive, used by 3+ items now."

Pectra introduces churn-paced exit and consolidation to replace Phase0's per-validator counter (`get_validator_churn_limit`). Instead of "max N validators exit per epoch," Pectra meters by balance: "max X gwei of exit per epoch." The two primitives are structurally identical (differing only in the churn-limit selector): both compute `earliest_*_epoch = max(state.earliest_*_epoch, compute_activation_exit_epoch(current))`, detect new-epoch with strict `<`, reset full budget on new epoch (or carry over `state.*_balance_to_consume` on same epoch), then if the request doesn't fit compute additional epochs via the **ceiling-division formula `(balance_to_process - 1) // per_epoch_churn + 1`**, then mutate state and return the exit/consolidation epoch.
